## Spring과 SpringBoot의 차이
스프링 프레임워크 : Java 기반 애플리케이션 개발을 지원하는 오픈소스 프레임워크, 스프링이라고도 불림.
스프링 프레임워크는 모듈화된 구조로 구성되어 있음. 

스프링의 주요 특징
- 트랜잭션 관리
- 데이터 엑세스
- 메세징
- 웹 애플리케이션 개발을 위한 MVC구조

고도로 구성된 애플리케이션을 개발할 수 있었으나, 이로 인해 스프링 프레임워크는 설정과 구성이 복잡하고 시간이 많이 걸림.

'스프링부트'는 이러한 복잡한 설정을 간소화해주고자 등장했음.
스프링부트의 목적은 바로 스프링 애플리케이션을 설정 없는 방식으로 쉽게 시작할 수 있게 하자였음.

즉, 초기 설정의 복잡성을 줄이기 위해 만들어진게 스프링부트.


### 정의와 목적
- Spring : Spring Framework는 엔터프라이즈 애플리케이션 개발을 위한 포괄적인 프레임워크이다. Dependency Injection(DI)와 Aspect-Oriented Programming(AOP) 등을 포함하여 다양한 기능을 제공함. Spring의 다양한 모듈(Spring MVC, Spring Data, Spring Security)등을 통해 애플리케이션 여러 레이어에서 활용이 가능함.

- SpringBoot : SpringFramework를 기반으로 한 '서브 프레임워크', Spring 애플리케이션을 더 쉽게 설정하고 시작할 수 있도록 도움. 기본적인 설정을 자동으로 처리해주기 때문에 개발자가 최소한의 설정으로 애플리케이션을 빠르게 개발하고 배포할 수 있습니다.
