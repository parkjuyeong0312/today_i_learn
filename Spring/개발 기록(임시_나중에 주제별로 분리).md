
# 빌드 도구
빌드 도구란?
소스코드를 실행가능한 애플리케이션으로 바꿔주는 과정
빌드는 컴파일, 패키징, 테스팅 등의 과정을 포함한다.
또한, 라이브러리 버전관리, 의존성관리 등의 기능을 제공한다.

사용하는 이유?
사용하는 이유로는 대규모 프로젝트에서, 빌드를 어떤순서로하고, 테스트를 하고, 또 의존성은 관리는 따로 하고, 이런게 너무 복잡하니까
한번의 과정으로 처리해주어서 편리하게 된다.

## Gradle과 Maven?
빌드도구에는 대표적으로 두가지가 있는데, 나중에 알아봐야겠다.

## Gradle 구성요소
- gradlew, gradlew.bat
    - gradlew는 gradle을 설치하지 않아도, 프로젝트를 빌드할 수 있게 해주는 **스크립트 파일**이다.
    - gradlew는 MAC, 리눅스 환경이고, gradlew.bat은 Window환경이다.
    - 완벽하게 설치한다기보다는 임시 저장소인 캐시에 두고, 꺼내쓰는 방식으로 작동함.
    - 해당 프로젝트 내에서만 작동하는 것임.

- gradle/wrapper 폴더
    - gradlew 스크립트가 작동할 수 있게 필요한 파일들이 있는 폴더
        - gradle-wrapper.jar: Gradle을 다운로드하고 실행하는 실제 자바 실행 파일
        - gradle-wrapper.properties : 다운로드할 gradle의 버전 정보들이 적혀있는 파일
- settings.gradle
    - 프로젝트의 전체적인 구성 정보를 담고 있는 파일
    - 프로젝트의 이름을 지정하고, 큰 프로젝트라면, 어떤 모듈들이 이 프로젝트에 포함되는지 정의함.
- build.gradle
    - 핵심 빌드 스크립트
    - 의존성
        - 코드 묶음인 라이브러리들을 땡겨옴.
    - 플러그인
        - 자바 컴파일, 스프링 부트 등, 실행에 돕는 플러그인들을 선언함.
    - 작업
        - 컴파일
        - 패키징
        - 테스팅
