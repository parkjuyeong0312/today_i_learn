# Annotation

### Lombok이란?
어노테이션 기반으로, 코드를 자동 완성해주는 **라이브러리**
Getter, Setter, Equals, ToString등과 같은 코드를 자동완성 시켜줌.
코드 다이어트를 할 수 있다.

#### @AllArgsConstructor
모든 변수를 사용하는 생성자를 자동완성시켜주는 어노테이션

#### @NoArgsConstructor
기본생성자를 자동완성 시켜주는 어노테이션

#### @RequiredArgsConstructor
특정 변수만을 활용하는 생성자를 자동완성 시켜주는 어노테이션  
생성자의 인자로 추가할 변수에 @NonNull 어노테이션을 붙혀서 해당 변수를 생성자의 인자로 추가할 수 있다.  
아니면 final로 선언해도, 의존성을 주입 받을 수 있다.

#### @Data
@ToString, @EqualsAndHashCode, @Getter, @Setter, @RequiredArgsConstructor를 자동완성 시켜준다.
실무에서는 너무 무겁고, 객체의 경우 안정성을 지키기 때문에, @Data의 사용을 지양한다.

#### @EqualsAndHashCode
객체의 모든 필드를 대상으로 비교 로직을 만듭니다.
equals 함수와 hashCode 함수를 자동으로 생성해준다.

#### @Builder
복잡한 객체를 마치 조립하듯이 안전하고 편리하게 생성할 수 있게 도와주는 패턴(Builder Pattern)을 자동으로 구현해주는 어노테이션  
생성자를 직접 사용하는 방식을 해결해주기 때문에, 좋음.
- 빌더는 쉽게 객체를 만들게 인터페이스를 제공하지만, 최종적으로는 생성자를 호출해야한다.
- 따라서, 클래스 레벨에서 @Builder를 호출하게 되면, 전체 필드에 대한 생성자가 생기게 되는거고,
그럴려면 @AllArgsConstructor가 필요하게 된다.
- 기본 생성자도 필요하다고 하면, @NoArgsConstructor를 추가해야된다.
- 특정 생산자 위에 @builder를 붙히면, 해당 생성자에 대해서만 빌더를 사용할 수 있다.
