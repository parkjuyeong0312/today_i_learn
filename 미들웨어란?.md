# 미들웨어(Middleware)
- 중간에 위치하는 소프트웨어
- 서로 다른 애플리케이션, 데이터베이스 ,통신 프로토콜 사이에서 다리 역할을 하며 서로 소통할 수 있도록 돕는 서비스

### 미들웨어 역할
- 통신 중재 : 서로 다른 네트워크 프로토콜이나 운영체제 환경에서 데이터가 오갈 수 있게 함.
- 데이터 관리 : 데이터베이스에 접근하거나 분산된 데이터를 통합하는 과정을 단순화함.
- 부하 분산 : 서버에 요청이 몰릴 때 적절히 분산시켜 시스템 안정성을 유지함.
- 보안 및 인증 : 사용자 인증이나 권한 확인 같은 보안 기능 처리.

### 미들웨어의 종류
- Was(Web Application Server)
    - 웹 환경에서 동적인 콘텐츠(비즈니스 로직)을 처리
    - Tomcat, Jeus
- RPC(Remote Procedure Call)
    - 원격에 있는 함수나 프로시저를 로컬처럼 호출하게함.
- MOM(Message Oriented Middleware)
    - 메세지 기반의 비동기 통신을 지원함.
    - RabbitMQ, Kafka
- DB 미들웨어
    - 애플리케이션과 데이터베이스 서버 사이에서 연결을 관리함.
    - ODBC, JDBC
- API Gateway
    - 클라이언트의 요청을 받아 적절한 마이크로서비스로 라우팅함. 
    - Spring Cloud Gateway


### 미들웨어를 왜 쓰나요?
- 개발 효율성
    - 통신이나 보안 같은 복잡한 하위 계층 로직을 직접 구현할 필요없이 미들 웨어를 갖다쓰면 된다.
- 유연성
    - 시스템의 일부를 변경하거나 확장할 때 전체를 뜯어고치지지 않고 미들웨어 설정만으로 대응하기 유리하다.
- 표준화
    - 다양한 환경, 클라우드, 온프레미스 등에서 일관된 방식으로 데이터를 주고 받을 수 있다.
