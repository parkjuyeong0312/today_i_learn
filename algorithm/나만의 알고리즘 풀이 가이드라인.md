## 문제 풀이 요령
- 문제를 꼼꼼하게 읽는다.(잘못된 문제 이해는 시간을 많이 소모한다.)
- 시간복잡도를 미리 계산해야한다.
- 문제를 먼저 풀지 않기. 주석으로, 전체적인 풀이를 점검 후에 구현에 들어간다.
- 변수는 명확하게 설정하는 것이 좋고, 그렇지 않다면, 주석으로 변수의 의미를 작성한다.

## 디버깅 요령
- 작성완료 후 정답이 아니라면 다음의 경우이다. 차례대로 점검해보자.
    - 논리가 틀렸다.
    - 매개변수가 잘못 들어가있다.
    - 특정 코드의 위치가 잘못되어있다.

## 코딩테스트 요령
- 나중에 코드 디버깅을 하기엔 너무어렵다. 따라서, 구현 후에 바로 실행하여, 무엇이 틀렸는지 확인해볼 것.





## 알고리즘 주제 별 풀이


### 이분탐색
#### 알고리즘 선택 기준
- 탐색 범위의 값이 비정상적으로 클 때 사용한다.
    - 이분탐색은 logN의 시간복잡도를 갖는다.
- 이분탐색의 조건은 데이터가 **정렬** 되어 있어야 한다는 것이다.
    - 데이터를 정렬해도 정답에 지장이 없다면, 정렬을 사용한다.
- 특정 지점을 기준으로(범위로), 조건의 만족 여부가 나뉠 때
    - lower bound, upper bound 를 사용

#### 풀이 방법
1. 정렬
2. 풀이
- left = 0, right = arr.length-1
- while(left<=right)
- mid = left+right/2
- if mid < target
    - left = mid+1
    - right = mid -1 
- upper bound, lower bound는 따로 설정해줘야함.
- 경계값의 조건은 문제를 보고 파악할 것.
