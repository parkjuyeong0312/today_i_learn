# 유니온 파인드 (Union-Find)

## 개념
`union`과 `find` 두 함수를 기반으로, **여러 노드가 같은 그룹(집합)에 속하는지 판별하고 합치는** 알고리즘이다.
서로소 집합(Disjoint Set)이라고도 불린다.

---

## 초기화
모든 노드는 자기 자신을 부모(루트)로 갖는다.

```java
for (int i = 0; i <= n; i++) {
    parent[i] = i;
}
```

---

## find — 루트 탐색 (경로 압축)

특정 노드의 루트를 찾는 함수다.
단순 재귀로 구현하면 트리가 일렬로 늘어질 때 `O(N)`이 걸리지만,
**경로 압축(Path Compression)** 을 적용하면 사실상 `O(1)`에 수렴한다.

```java
int find(int x) {
    if (parent[x] == x) return x;       // 루트에 도달하면 반환
    return parent[x] = find(parent[x]); // 루트를 직접 부모로 갱신 (경로 압축)
}
```

> **경로 압축 원리**
> 재귀 호출이 끝나며 돌아올 때, 거쳐온 모든 노드의 부모를 최종 루트로 직접 연결한다.
> 이후 같은 노드에 접근하면 바로 루트를 반환할 수 있다.

---

## union — 두 집합 합치기

두 노드 A, B가 서로 다른 집합이면 하나로 합친다.
각자의 루트를 찾아서 한쪽 루트를 다른 쪽의 부모로 연결한다.

```java
void union(int a, int b) {
    a = find(a);
    b = find(b);
    if (a != b) {
        parent[b] = a; // b의 루트를 a 아래로 편입
    }
}
```

---

## 같은 집합 판별

```java
boolean isSame(int a, int b) {
    return find(a) == find(b);
}
```

---

## 시간 복잡도

| 연산 | 경로 압축 적용 시 |
|------|-----------------|
| find | O(α(N)) ≈ O(1) |
| union | O(α(N)) ≈ O(1) |

> `α(N)`은 아커만 함수의 역함수로, 사실상 모든 입력에서 4 이하다.
