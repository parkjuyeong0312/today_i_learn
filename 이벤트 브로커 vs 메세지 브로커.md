# 메세지브로커 vs 이벤트 브로커

## 메세지 브로커
- 서비스 간의 메시지 전달을 중개
- 서비스 간의 결합도를 낮추고, 메세지 전달의 신뢰성을 높이는 데 도움을 줌.
- 장애 발생시 메세지 손실 방지, 순서 보장 등의 기능 제공
- 데이터의 수명이 짧습음.
- 서로 다른 시스템 간의 통신(비동기 처리)에 집중함.
예 : RabbitMQ 


## 이벤트 브로커
- 이벤트를 게시하고 구독하는 역할
- 대용량의 데이터를 실시간을 처리할 수 있음.
- 데이터 영구 저장 기능을 제공함.
- 메세지 브로커의 기능을 포함하며, 이벤트를 저장하는 능력 추가 된 상태임.
예 : Apache Kafka
