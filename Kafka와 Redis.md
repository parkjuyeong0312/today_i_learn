# Kafka란?
![alt text](images/kafka%20logo.png)
- 대용량 데이터를 실시간으로 전달하고 처리하는 분산 메세징 플랫폼
- Pub-Sub 모델을 따름.
    - Pub-Sub 모델에서 Borker의 역할을 맡음.
- 주로 기업에서 대규모 데이터 처리 및 이벤트 기반 시스템을 구축하는데 사용함.

### Kafka를 사용하는 이유
- 높은 처리량 : 짧은 시간에 대량의 데이터를 지연 없이 처리할 수 있어서 빅데이터 시스템에서 필수적임.
- 확장성 : 데이터 양이 늘어나면, 서버(Broker)를 추가하기만 하면 됨.
- 영속성 : 메세지를 메모리가 아닌 디스크에 저장함. 소비자가 데이터를 가져간 후에도 데이터가 사라지지 않고 설정된 기간 동안 보관됨.(장애 복구에 유리함)
- 분산 처리 : 데이터를 여러 서버에 나누어 저장하므로, 한 대의 서버가 고장나도 시스템이 안멈춤.
