# 빌드 도구란?
개발자가 작성한 소스코드를 실행 가능한 애플리케이션으로 만들어주는 자동화 프로그램
- 의존성(Dependency)관리
    - 프로젝트를 개발할 때, 외부 라이브러리들을 인터넷에서 다운로드하고 버전을 맞춰줌.
- 컴파일(Compile)
    - 사람이 읽을 수 있는 소스 코드를 컴퓨터가 이해할 수 있는 언어(바이트코드 등)로 변환함.
- 테스트 자동화
    - 코드가 정상적으로 작동하는지, 테스트 코드가 있다면, 빌드 과정중에 실행해줌
  
예시 : Gradle, Maven

# 패키징이란?
빌드 과정의 마무리 단계, 컴파일 된 코드와 외부 라이브러리, 설정 파일 등을 어디서나 쉽게 실행할 수 있게 하나의 덩어리로 묶어주는 작업.

예시 : .jar, .war

<전체적인 개발 및 배포 파이프라인>
코드 작성 ➔ 빌드 도구로 컴파일 및 테스트 ➔ .jar 등으로 패키징 ➔ 도커 이미지로 생성 ➔ 도커 컨테이너로 실행

# 라이브러리란?
소프트웨어 개발용 코드 묶음
