# 도커란?
- Go언어로 작성된, 리눅스 컨테이너 기반으로 하는 오픈소스 가상화 플랫폼
- docker에 설치하면 나중에 지울 때 편리하게 지울 수 있음.
- 특정 프로그램을 설치하고 관리하기가 편리해진다.

### 도커 이미지
- 실행 파일 모음, 설계도
- 상태가 변하지 않음(Read-only)
- 애플리케이션에 필요한 코드, 런타임, 라이브러리, 환경변수 등이 포함되어있음.

### 컨테이너 
- 이미지가 실제로 실행된 상태
- 이미지를 바탕으로, 독립된 공간에서 격리되어 실행됨. 컨테이너에서 어떤 행동을 하든 이미지에는 영향을 못줌

### 도커 레지스트리
- 도커 허브

### 도커가 좋은 이유?
- 기존에 윈도우에 리눅스를 돌리러면 VM을 써야했다.
    - VM은 하드웨어 전체를 가상화, OS가 통째로 들어가야해서 무겁다.
- Docker는 호스트의 OS 커널을 공유하며, **프로세스만 격리함.**
- 따라서 메모리도 적게 먹음


### 이미지를 만드는 법
- docker file을 작성하면됨.
- 이 파일을 빌드하면, 도커 이미지가 생성됨.

### 도커 컴포즈?
- 요즘 서비스는 데이터베이스, 웹서버 하나 등 여러개의 컨테이너가 함께 움직어야됨.
- 이걸 하나로 묶어서 관리해주는 도구가 Docker compose임.
- docker-compose.yml에 파일 설정을 적어두면 명령어 한번으로 전체 시스템을 껏다 켰다 할 수 있음.

## 명령어 정리
> docker run [옵션] [이미지_이름] [컨테이너_내부에서_실행할_명령어]
- docker run
    - 새로운 도커 컨테이너를 만들고 실행하는 명령
- --name (이름)
    - 새로 띄울 컨테이너의 이름을 지정.
- -d
    - 컨테이너를 백그라운드에서 실행함.
    - 이 옵션이 없으면, 터미널 화면에 MySQL로그가 계속 찍히고, 터미널을 끄면 DB도 꺼짐.
- -p 3306:3306
    - 포트번호를 연결 해주는 옵션
    - [내 컴퓨터 포트] : [도커 컨테이너 내부 포트]
- -e (내용)
    - 컨테이너 내부에 환경변수 설정
- [이미지_이름]
    - 로컬에 있는 이미지가 아니라면, 도커 허브(docker hub)의 이미지 저장소에서, 다운로드함.
